Data I/O¶

MusPy provides three type of data I/O interface.

  • Reading and writing with MIDI and MusicXML

  • Saving and loading with JSON and YAML

  • Conversion between objects in other packages

Reading and Writing¶

MusPy supports reading and writing with MIDI nad MusicXML formats.

  • MIDI: muspy.read_midi() and muspy.write_midi()

  • MusicXML: muspy.read_musicxml() and muspy.write_musicxml()

Saving and Loading¶

MusPy supports save/load functions for losslessly storing Music object.

  • JSON: muspy.save_json() and muspy.load_json()

  • YAML: muspy.save_yaml() and muspy.load_yaml()

Hint

An example of a MusPy Music object saved as a YAML file is available here.

JSON/YAML Schemas¶

MusPy provides schemas in JSON and YAML formats. These can be used to validate a JSON or YAML file for the Music object.

Interface with other packages¶

MusPy supports conversion between muspy.Music objects and objects from other packages, such as pretty_midi.PrettyMIDI and pypianoroll.Multitrack objects.

  • pretty_midi: muspy.from_pretty_midi() and muspy.to_pretty_midi()

  • Pypianoroll: muspy.from_pypianoroll() and muspy.to_pypianoroll()
