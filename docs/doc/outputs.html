muspy.outputs¶

Output Interfaces¶

MusPy provides the following output interfaces.

  • Save losslessly to JSON or YAML: muspy.save_json() and muspy.save_yaml()

  • Write to other formats: muspy.write_midi() and muspy.write_musicxml()

  • Convert to other objects: muspy.to_pretty_midi() and muspy.to_pypianoroll()

  • Wrappers: muspy.save(), muspy.write() and muspy.to_object()

muspy.outputs.save(music, path, kind=None, **kwargs)[source]¶

Save a Music object loselessly to a JSON or a YAML file.

Parameters

  • music (muspy.Music object) – MusPy Music object to be saved.

  • path (str or Path) – Path to save the file.

  • kind ({'json', 'yaml'}, optional) – Format to save. If None, infer the format from the extension of path.

See also

muspy.write()

Write to other formats such as MIDI and MusicXML.

Notes

The conversion can be lossy if any nonserializable object is used (for example, in an Annotation object, which can store data of any type).

muspy.outputs.to_event_representation(music, min_step=1, **kwargs)[source]¶

Return a Music object in event-based representation.

Parameters

  • music (muspy.Music) – MusPy Music object to be converted.

  • min_step(optional) – minimum quantification step decide how many ticks to be the basic unit (default = 1)

  • tick_dim(optional) – tick-shift event dimensions the maximum number of tick-shift (default = 100)

  • velocity_dim(optional) – velocity event dimensions the maximum number of quantized velocity (default = 32, max = 128)

Returns

  • array (numpy.ndarray) – Converted event-based representation. size: L * D: - L for the sequence (event) length - D = 1 {

    0-127: note-on event, 128-255: note-off event, 256-355(default):

    tick-shift event 256 for one tick, 355 for 100 ticks the maximum number of tick-shift can be specified

    356-388 (default):

    velocity event the maximum number of quantized velocity can be specified

    }

  • e.g.

  • [C5 - - - E5 - - / G5 - - / /]

  • ->

  • [380, 60, 259, 188, 64, 258, 192, 256, 67, 258, 195, 257]

muspy.outputs.to_monotoken_representation(music, min_step=1)[source]¶

Return a Music object in monotoken-based representation.

Parameters

  • music (muspy.Music) – MusPy Music object to be converted.

  • min_step(optional) – minimum quantification step decide how many ticks to be the basic unit (default = 1)

Returns

array – Converted monotoken-based representation. size: L * D:

  • L for the sequence length

  • D = 1 {

    0-127: pitch onset 128: hold state 129: rest state

}

e.g.

[C5 - - - E5 - - / G5 - - / /] -> [60, 128, 128, 128, 64, 128, 128, 129, 67, 128, 128, 129, 129]

Return type

muspy.outputs.to_note_representation(music, min_step=1)[source]¶

Return a Music object in note-based representation.

Parameters

  • music (muspy.Music) – MusPy Music object to be converted.

  • min_step(optional) – minimum quantification step decide how many ticks to be the basic unit (default = 1)

Returns

array – Converted note-based representation. size: L * D:

  • L for the sequence (note) length

  • D = 4 {

    pitch: 0 - 127, start: start ticks, end: end ticks, velocity: 0 - 100

}

e.g. [C5 - - - E5 - - / G5 - - / /] -> [[60, 0, 3, 100], [64, 3, 5, 100], [67, 6, 8, 100]] (assume velocity to be 100)

Return type

numpy.ndarray
