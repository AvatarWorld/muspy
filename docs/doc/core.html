Core Functions¶

Core functions¶

Core functions that can be applied to a muspy.Music object.

muspy.core.append(obj1, obj2)[source]¶

Append an object to the correseponding list.

If obj1 is of type muspy.Music, obj2 can be muspy.TimeSignature, muspy.KeySignature, muspy.Tempo, muspy.Lyric, muspy.Annotation or muspy.Track. If obj1 is of type muspy.Track, obj2 can be muspy.Note, muspy.Lyric or muspy.Annotation.

Parameters

obj (MusPy objects (see above)) – Object to be appended.

muspy.core.clip(obj, lower=0, upper=127)[source]¶

Clip the velocity of each note.

Parameters

  • obj (muspy.Music, muspy.Track or muspy.Note) –

  • object – Object to be clipped.

  • lower (int or float, optional) – Lower bound. Defaults to 0.

  • upper (int or float, optional) – Upper bound. Defaults to 127.

Quantize all the time-stamped objects, assuming absolute timing.

Parameters

  • music (muspy.Music object) – Object to be quantized.

  • step (int or float) – Length of quantization step, in seconds.

  • beat_resolution (int) – Time steps per beat.

Parameters

  • step (np.ndarray or list of int or float) – Length of quantization step, in seconds.

muspy.core.sort(obj)[source]¶

Sort all the time-stamped objects with respect to event time.

If a muspy.Music is given, this will sort time signatures, key signatures, tempos, lyrics and annotations, along with notes, lyrics and annotations for each track. If a muspy.Track is given, this will sort notes, lyrics and annotations.

Parameters

obj (muspy.Music or muspy.Track object) – Object to be sorted.

muspy.core.transpose(obj, semitone)[source]¶

Transpose all the notes by a number of semitones.

Parameters

  • obj (muspy.Music, muspy.Track or muspy.Note) –

  • object – Object to be transposed.

  • semitone (int) – The number of semitones to transpose the notes. A positive value raises the pitches, while a negative value lowers the pitches.
